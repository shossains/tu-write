Two rules for recommendations

Rule 1

Present your recommendations directly after the conclusion. If you have many recommendations, you can discuss them in a proper chapter. If you only have a few recommendations to give, it is best to place them in one chapter with the conclusion. Discuss them in a final paragraph and tell how many recommendations you have. You can also add a heading. In this way, your recommendations can be read as a stand-alone document.

Rule 2

Recommendations have to be feasible, appropriate, specific and encouraging.

Challenge

In this challenge you are asked to assess four pairs of recommendations. Each pair presents one of the features (see rule 2).

The recommendations have been extracted from reports written by first year students Computer Science. The students  have to write the code for a bot team that is able to independently win StarCraft, a computer game. The report is about the building process and the results. The recommendations are meant for those who have to do the same project next year. 

See if you can find the recommendation that meets the feature best.
